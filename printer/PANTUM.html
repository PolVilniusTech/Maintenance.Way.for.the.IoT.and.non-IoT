Web Admin Panel

Access to Web Admin Panel Available from Wired Network and Wireless Network Side. Wireless Network by default are set to ”WPA2 Personal”. Selection for ”No Authentication” is Available in the Settings.

Web Admin Panel (or Control Panel Interface) does not seem to contain any End User License Agreement and (or) the Privacy Policy. [EULA] [Privacy]

Web Admin Panel (following - CPI) could be accessed using the Web Browser who are able to process the HTML4 + JQuery (JS based). By turning off the JS probably effective connectivity and (or) control with the CPI would be cut off. [HTML]

CPI displays the copyright from the Year of 2015.

CPI could be accessed by HTTP and by HTTPS (Self-signed Certificate from 2019-09-11 EES). [HTTP and HTTPS at the same time] OpenSSL library are used for generation of Certs. No Licenses Available of third party libraries. [No Licenses]

CPI Only allows single Admin Account with username as ”admin”. No additional Security Options shown for protecting the Admin Account. [Single Login] Input Security - data input are controled-cleaned with help of Dynamic JS. Idea for the Consumer by himself to be able specify further-additional constraints, i.e. if Phone No. has Protection of ”...”, then Consumer may wish to make it more strict to specific format.

JS in CPI does not have unified-consistant checks for end user (or Consumer) ”Browser” Program, i.e. ”navigator.appName;” ”navigator.appVersion;” ”navigator.userAgent.toLowerCase().indexOf(’safari’)”

By look at Ref. https://www.w3schools.com/js/js˙window˙navigator.asp best option would be to check ”navigator.userAgent” and get all the needed data from there. When there exists Internet full of Open Source Browser Apps, the data in ”navigator.appName” and ”navigator.appVersion” are not releable Option at all, unless if there would be some ISO Standard who would link specific Company Program to Specific App Name. Nowadays we have a lot of Standard Offering Companies too, so more easiest example could be Specific Community maintained Database which would have/has support by most of specific ”Browser” App Creators.

JS code would be more safe, if there would be centralised checks like in previous reference, i.e.: navigator.cookieEnabled; navigator.javaEnabled(); navigator.onLine; what about other technologies/pre-processors/current status?

Strange Things would be to find something like following (in JS code): ”[S(”“WEP1“”), 0]” ”[S(”“WEP2“”), 1]” ”[S(”“WEP3“”), 2]” ”[S(”“WEP4“”), 3]” Parallel understanding via different languages (EN, Chinese...) and security functionality and technologies in Practice are in Question? WEP is less secure than WPA, WPA2. WEP is more secure that No Authentication. WEP currently could not be selected via CPI. If there were official WEP update in more secure option, then I would wish to have for such knowledge.

CPI is Powered by jQuery of First Major version 1.X. According ref. https://releases.jquery.com/jquery/ there are already Third Major version 3.X [jQuery]

CPI from Admin Settings allows to reset CPI into Factory Defaults.

Wired Network in automatic fashion sets up only IPv4, but there should be way to set up IPv6 too. [Set Up of IPv6]

Each Query of CPI expires in the Past as ”Thu, 01 Jan 1970 00:00:00 GMT”. Server in Responses provides Cache Setting as ”no-cache” for the Browser to comply. [Bad Practice]

CPI does not use Security Measures like HTTP Strict Transport Security, Public Key Pinning, Content Security Policy. [Security Headers]

CPI are providing twelve (12) choises for language selection. Idea for the Consumer to provide these languages as set of two values where first would be shown in Original language as in current situation and for second one - according default locale and (or) some Admin Setting. [Idea]

CPI has good Security Option for letting Consumers to prolong their CPI’s Security of the HTTPS protocol by generating new Self-signed Certs. If we are taking in the Account that Self-signed Certificate valid not earlier than 2019-09-11 EES, then after around up to 4-5 Years from this date using ”Public-Key: (2048 bit)” Certificate would be good to conduct-issue new Cert. Unfortunately there no Option to choose ”Public-Key: (4096 bit)” which could be valid a bit longer. [OpenSSL]
