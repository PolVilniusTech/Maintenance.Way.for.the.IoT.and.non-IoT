3 E5573Cs router Admin Panel

3.1 Web Admin Panel

Access to Web Admin Panel Available from Wireless Network Side Only.
Wireless Network by default are set to ”WPA2-PSK”. Selection for ”None (Open)” is Available in the WLAN Basic Settings. The Option of ”WEP” is Available too.
Web Admin Panel instead of helpful information per menu provides Professional Mobile WiFi ”Help” User Manual.
Web Admin Panel provides Statistical information about Data Stream Volume per Connection, Monthly Data and Total Data. It also let us see the Time for how long it were Active. From (Authenticated) User Input there is possible to define ”Start date” (wrong), ”Monthly data plan” (ok), Threshold (ok). From Documentation perspective it sounds that ”Threshold” controls only appearance in the ”Statistic” Page. The ”Start date” represents the single day of the Month from when the Data Plan should be start to do actual counting.
It’s cool that Session Identificator for the Web Admin Panel is set to ”httpOnly” and after specific Time Users are getting log out for being idle or by inactivity. The problem with ”httpOnly” attribute is when You start Sniffing the Data Traffic there would be Session Identificator presented unencrypted. Some tech could not even know about such attribute as ”httpOnly”, so when using HTTPS it still would be provided unencrypted. In other hand Web Admin Panel does not provide possibility to the HTTPS in the first place by responding ”Unable to connect” for URL: the Home Page
Web Admin Panel has manual ”Update” way. In the menu ”Update” it shows ”Software version” and ”Web UI version” of the instrument E5573Cs. It would be cooler than that, if in addition of this information alongside would provide information which is Available from Online Sources, so You would not need to Click the Button ”Check for updates”, if there is no reason for that. In other hand there should exist dynamic notification in the Web Page, if the Update from the Official API Source is Available.
Web Admin Panel Login Form accepts only password with ”maxlength” of 15 (fifteen). This is the Problem. Nowdays of course would be more wise to suggest authentication using Ubikeys or some sort of certificate authentication using Your own Storage Device.
From HTML perspective there are compliance with no less than HTML4 version. No ”title” HTML tag defined. There are some CSRF tokens who protects the device from negative effects from the ”outside” while People are surfing through the Net.
Web Admin Panel are using 1.7.2 jQuery (2012-03-21). According the Year which are provided in the User Guide ”Copyright” statement of 2018 there should be already 3.3 Version of jQuery Available.
From Web Admin Panel it could be assumed that Support library for Security ”OpenSSL” is not used. From the Year of 2022 ”OpenSSL” already has 3rd Major version. All Protective means in Web Admin Panel are implemented using JavaScript.
